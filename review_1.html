Classificação versus regressão

O aluno identifica corretamente o tipo de estimativa que o problema requer e a justifica de maneira razoável.

Correto! Mais especificamente, é um problema de classificação não por causa do tipo da saída e sim por causa do tipo da variável alvo (passed no nosso caso). Por exemplo, além de classes, nós podemos também ter probabilidades como saída, no qual é uma variável contínua (usando predict_proba no sklearn). O problema é ainda de classificação uma vez que a variável alvo (passed) contém valores discretos (binários).

Excelente

Ótimo trabalho por utilizar estratificação aqui! Conforme vimos anteriormente em grad_rate, esta base de dados é um pouco desbalanceada. 67% dos estudantes passaram e 33% reprovaram. Idealmente, esta proporção deve ser preservada para os novos subconjuntos de treino e teste. Durante o processo de divisão treino/teste cada valor de random_state acaba criando uma proporção diferente. A estratificação permite que esta proporção seja preservada. Desta forma, a apresentação de resultados será mais consistente com a realidade e teremos maior confiança sobre a influência de parâmetros no modelo final. A amostragem continua sendo aleatória, no entanto com a proporção de classes preservada.

Treinando e avaliando modelos

Três modelos supervisionados são escolhidos e razoavelmente justificados. Prós e contras de cada modelo são fornecidos, além de uma discussão de aplicações gerais de cada.

Sobre suas escolhas:

  • SVM é bastante poderoso graças ao ‘kernel trick’ para criar fronteiras de decisão não lineares. No entanto é um estimador bastante lento. Como alternativa há o Stochastic Gradient Descent no qual implementa uma versão aproximada de Linear SVM e Logistic Regression. Como vantagens é mais rápido e suporta online learning. No entanto, não suporta o ‘kernel trick’ e seu resultado é aproximado.
  • Regressão Logística é um método bastante simples e poderoso. Muito utilizado em bancos para análise de risco de crédito. Uma de suas principais vantagens é o fato de também fundamentalmente retornar probabilidades como saída. No entanto, o mesmo é limitado à fronteiras de decisões lineares. Pode ser um ótimo ponto de partida para comparar com estimadores mais complexos.
  • Árvore de decisão é um dos estimadores populares. Muito utilizado na medicina devido à sua simplicidade e abordagem 'caixa branca' no qual podemos visualizar como as decisões funcionam em uma estrutura semelhante a uma árvore. No entanto, como desvantagens, este estimador tem uma tendência indesejada de se sobreajustar (overfitting) aos dados de treinamento. Outra desvantagem é que as fronteiras de decisão são alinhados aos eixos: portanto, fronteiras de decisão que são 45 graus em relação aos eixos precisariam de um grande número de eixos paralelos sendo assim um resultado subótimo. Como alternativa ao problema de overfitting, podemos usar ensembles de árvores de decisão, como Random Forest ou algoritmos do tipo boosting. Atualmente, eles são muito populares em competições de ciência de dados.

Sugestão

Apesar de não ser exigido neste projeto, são boas práticas normalizar ou escalar seus atributos antes de utilizá-los como entrada no modelo. Isso é importante em especial para modelos como SVM e LR. sklearn.preprocessing tem várias soluções para este propósito.

Comentário

Você pode criar uma lista de dicionários de parâmetros para evitar combinações de parâmetros não consistentes (por exemplo, o parâmetro 'degree' somente tem influência quando o kernel 'poly' é utilizado, já o parâmetro 'gamma' não está disponível para o kernel 'linear'). Para ter uma combinação de parâmetros mais consistente, você pode criar uma lista de dicionários de parâmetros. Segue um exemplo:

parameters = [
    {'C': [0.1, 1, 10], 'kernel': ['linear']},
    {'C': [0.1, 1, 10], 'degree':[1, 2, 3], 'kernel': ['poly']},
    {'C': [0.1, 1, 10], 'gamma': [0.5, 0.1, 0.01], 'kernel': ['rbf']}
]

A pontuação F1 é calculada para o modelo calibrado e tem desempenho igual ou melhor ao do modelo escolhido com as configurações padrão.

A pontuação no conjunto de treinamento é de 0.8664 e no de testes 0.8052, não houve mudanças na pontuação após a calibração.

Recomendaria que acrescentasse mais hiperparâmetros para serem calibrados, em especial mais valores para o parâmetro C.

import seaborn as sns # Install using 'pip install seaborn'

cm_test = confusion_matrix(y_test, clf.predict(X_test))

sns.heatmap(cm_test, annot=True, cmap='Greys', xticklabels=['No', 'Yes'], yticklabels=['No', 'Yes'])

plt.ylabel('True')
plt.xlabel('Predicted')
